# Workspace Manager 开发任务拆解 v1

## 1. 目标

- 以 MVP 为目标分阶段推进 `Workspace Manager`
- 先完成最小可运行桌面工具，再逐步增强稳定性与自动化能力

## 2. 里程碑概览

### M0：项目基础搭建
- 初始化解决方案与项目分层
- 建立文档、目录、测试工程
- 约定领域模型与服务接口

### M1：桌面基础控制闭环
- 完成托盘常驻
- 完成桌面图标显示/隐藏
- 完成全局快捷键注册

### M2：布局快照闭环
- 完成布局读取
- 完成布局保存
- 完成布局恢复与基础校验

### M3：模式与整理 MVP
- 完成模式切换
- 完成手动整理
- 完成规则配置与本地存储

### M4：可用性与稳定性增强
- 完成自动整理
- 完成撤销机制
- 完成演示模式增强

## 3. 详细任务拆解

## 3.1 M0：项目基础搭建

### 交付物

- 工程目录
- 协作文档

## 3.2 M1：桌面基础控制闭环

### 目标
- 能通过托盘或快捷键切换桌面图标显示状态

### 任务
- [ ] 实现托盘图标初始化与菜单
- [ ] 添加“显示/隐藏桌面图标”命令
- [ ] 建立 `DesktopIconService` 抽象与 Interop 实现
- [ ] 验证 Windows 句柄查找与消息发送逻辑
- [ ] 实现全局快捷键注册、注销与冲突提示

### 验收标准
- [ ] 托盘菜单可触发显示/隐藏动作
- [ ] 快捷键可稳定切换
- [ ] 失败时有日志与提示

## 3.3 M2：布局快照闭环

### 目标
- 能保存当前桌面图标布局并恢复

### 任务
- [ ] 调研并实现桌面图标位置读取
- [ ] 定义布局快照数据结构
- [ ] 实现布局保存到 `data/layouts/`
- [ ] 实现布局恢复
- [ ] 增加恢复前冲突校验
- [ ] 增加分辨率变化兼容策略

### 验收标准
- [ ] 可保存至少 2 套布局
- [ ] 恢复后图标位置基本一致
- [ ] 异常项有日志记录

### 目标

### 任务

### 验收标准
- [ ] 工作、默认、演示模式可切换
- [ ] 一键整理可执行且不删除文件

## 3.5 M4：可用性与稳定性增强

### 目标

### 任务
- [ ] 实现桌面目录监听
- [ ] 实现自动整理与延迟执行
- [ ] 实现最近一次整理撤销
- [ ] 完成演示模式恢复机制

### 验收标准

## 4. 建议实现顺序

- 第一步：先打通 `托盘 + 图标显示/隐藏`
- 第二步：再打通 `布局读取 + 布局恢复`
- 第三步：接入 `模式系统`
- 第四步：实现 `手动整理`
- 第五步：补 `自动整理 + 撤销`

## 5. 模块责任清单

### `App`
- 应用入口
- 宿主初始化
- 主窗口与托盘入口

### `UI`
- 页面与 ViewModel
- 设置与状态展示

### `Application`
- 服务接口
- 用例编排
- 模式切换流程

### `Domain`
- 领域模型
- 规则对象
- 布局对象

### `Infrastructure`
- 配置读写
- 文件系统操作
- 日志与持久化

### `Interop`
- Windows API 封装
- Explorer 与桌面交互
- 热键与自启动实现

## 7. 风险提示

- 桌面图标布局恢复是技术难点，建议尽早验证
- Explorer 相关句柄在不同系统环境下可能存在差异
- 自动整理要谨慎，必须优先保证可撤销与可追踪
- 测试工程依赖 NuGet 包，后续需要本机安装 .NET SDK 后再还原验证
