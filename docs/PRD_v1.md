# Windows 桌面管理器 PRD v1

## 1. 项目概述

### 1.1 项目名称
- 暂定名称：`Workspace Manager`
- 产品定位：面向 Windows 用户的轻量桌面管理工具

### 1.2 产品愿景
- 让 Windows 桌面从“临时堆放区”变成“可管理的工作区”
- 提供桌面状态切换、图标布局恢复、文件整理与演示场景支持

### 1.3 目标用户
- 日常桌面文件较多的办公用户
- 需要频繁录屏、演示、共享屏幕的用户
- 对桌面整洁度和效率有要求的个人用户

### 1.4 核心价值
- 一键切换桌面状态
- 保存并恢复桌面图标布局
- 自动或半自动整理桌面文件
- 以低打扰方式常驻系统托盘

## 2. 问题定义

### 2.1 当前痛点
- Windows 桌面图标的显示/隐藏入口较深，操作繁琐
- 桌面图标布局被打乱后难以恢复
- 桌面文件容易堆积，缺乏自动整理机制
- 演示、会议、录屏前需要手动清理桌面，操作低效且容易遗漏

### 2.2 目标结果
- 用户可以在 1 次操作内切换到目标桌面状态
- 用户可以在图标错乱后快速恢复既有布局
- 用户可以通过规则减少桌面文件堆积
- 用户可以在演示场景下快速获得干净、安全的桌面视图

## 3. 产品范围

### 3.1 首版本包含
- 托盘常驻
- 桌面图标显示/隐藏
- 全局快捷键
- 桌面布局保存/恢复
- 桌面模式切换
- 随机壁纸切换与定时轮换
- 一键整理桌面
- 基础规则管理
- 本地配置持久化

### 3.2 首版本不包含
- 云同步
- 跨平台支持
- 团队协作功能
- 替代 Explorer 的完整桌面壳层
- 复杂 AI 分类或自动推荐

## 4. 功能需求

## 4.1 托盘常驻

### 目标
- 让用户无需打开主窗口即可完成高频操作

### 功能点
- 应用启动后常驻系统托盘
- 左键支持快速触发默认动作
- 右键菜单提供常用入口：
  - 显示/隐藏桌面图标
  - 显示/隐藏任务栏
  - 切换模式
  - 保存布局
  - 恢复布局
  - 一键整理
  - 打开设置
  - 退出应用

### 验收标准
- 托盘图标稳定显示
- 菜单操作响应时间小于 500ms
- Explorer 重启后托盘能力可恢复或应用可自恢复

## 4.2 桌面图标显示/隐藏

### 目标
- 降低用户隐藏或恢复桌面图标的操作成本

### 功能点
- 支持从托盘一键切换桌面图标可见性
- 支持全局快捷键切换
- 支持启动时恢复上次状态或使用默认状态

### 验收标准
- 切换动作稳定成功
- 同一会话中状态一致且可追踪
- 不影响用户桌面文件真实存在状态

## 4.3 任务栏显示/隐藏

### 目标
- 让用户在演示、录屏或专注场景下快速收起任务栏

### 功能点
- 支持从托盘一键切换任务栏可见性
- 支持从主窗口快速切换任务栏
- 支持读取当前任务栏显示状态

### 验收标准
- 主任务栏与副任务栏可同时切换
- 切换结果在主窗口中可见
- 不影响其他桌面管理功能

## 4.4 桌面布局保存/恢复

### 目标
- 让用户在图标错乱、分辨率变化或误操作后快速恢复桌面秩序

### 功能点
- 保存当前桌面图标布局为快照
- 支持多个布局快照
- 每个布局快照显示保存时的桌面预览图
- 支持指定快照恢复
- 恢复前提示异常项：
  - 文件已不存在
  - 新增图标不在快照中
  - 分辨率变化导致坐标可能越界

### 验收标准
- 可以保存并读取多个布局
- 列表中可直观看到布局预览，便于区分相似快照
- 恢复后大多数图标位置与保存时一致
- 恢复失败时给出明确日志和提示

## 4.5 桌面模式切换

### 目标
- 让用户可在不同使用场景间快速切换

### 预设模式
- 默认模式
- 工作模式
- 演示模式
- 支持新增自定义模式
- 支持编辑和删除自定义模式

### 模式包含内容
- 桌面图标是否显示
- 任务栏是否显示
- 绑定的布局快照
- 是否启用整理规则
- 是否隐藏敏感项

### 验收标准
- 模式切换时间小于 1 秒
- 模式切换后状态与配置一致
- 提供默认、工作、演示三种预设模式
- 支持创建自定义模式，并与预设模式一起切换
- 自定义模式支持从模式页直接编辑、删除
- 新增自定义模式入口保持紧凑，不占用主界面长表单区域
- 默认模式可在设置中指定，并在启动时自动应用
- 演示模式退出时可恢复先前状态

## 4.6 手动整理桌面

### 目标
- 为用户提供低风险的一键整理入口

### 功能点
- 支持按类型整理
- 支持按日期整理
- 支持按规则整理
- 提供最近一次整理的撤销能力

### 验收标准
- 默认行为为移动，不删除文件
- 每次整理生成操作记录
- 撤销成功率高且路径可恢复

## 4.7 自动整理规则

### 目标
- 减少用户手动维护桌面的频率

### 功能点
- 监听桌面目录文件新增或变更
- 根据规则自动移动文件
- 支持延迟执行
- 支持白名单
- 支持启用/禁用单条规则
- 支持规则优先级

### 规则维度
- 扩展名
- 文件类型
- 文件名关键词
- 目标路径

### 验收标准
- 规则执行可预期
- 文件不会被重复移动
- 冲突规则有明确优先级行为

## 4.8 设置与配置

### 功能点
- 开机自启
- 启动后最小化到托盘
- 最小化时隐藏到托盘
- 关闭窗口时隐藏到托盘
- 默认模式设置
- 壁纸源配置、启动时切换与定时轮换
- 支持手动添加和删除自定义壁纸源
- 快捷键设置
- 布局管理
- 规则管理
- 配置导入导出
- 日志查看或导出入口

### 验收标准
- 所有设置可持久化
- 快捷键可在设置页自定义，并在保存后立即生效
- 当窗口缩到托盘且任务栏被隐藏时，仍可通过独立快捷键恢复主窗口
- 壁纸轮换可按已保存配置在后台持续执行，间隔支持按分钟设置
- 自定义壁纸源可在界面中维护并随本地配置持久化，内置图源会自动补齐
- 导入配置时有基本校验
- 不合法配置不会导致程序崩溃

## 4.9 日志与恢复

### 功能点
- 记录关键操作日志
- 记录自动整理结果
- 记录失败原因
- 支持导出日志

### 验收标准
- 用户可以定位关键失败原因
- 整理与恢复流程存在可追踪记录

## 5. 非功能需求

### 5.1 性能
- 应用冷启动目标小于 2 秒
- 常驻内存尽量控制在轻量级范围
- 文件监听不应造成明显 CPU 持续占用

### 5.2 稳定性
- Explorer 重启后不应导致核心能力永久失效
- 异常操作应记录并可恢复
- 后台常驻进程需避免频繁崩溃

### 5.3 安全性
- 默认不删除用户文件
- 任何高风险操作需确认
- 敏感模式切换需具备恢复能力

### 5.4 易用性
- 核心高频动作在两步内完成
- 默认配置应开箱可用
- 文案需明确区分“隐藏图标”和“移动文件”

## 7. MVP 定义

### MVP 目标
- 做出可真实日常使用的首个版本，而不是概念验证

### MVP 功能集合
- 托盘常驻
- 显示/隐藏桌面图标
- 全局快捷键
- 三种基础模式
- 布局保存/恢复
- 一键整理桌面
- 基础规则管理
- 配置持久化

### MVP 成功标准
- 用户能够连续使用 3 天而无需频繁手动维护桌面
- 核心功能无明显误移动、误恢复问题
- 演示场景下能快速得到干净桌面

## 8. 风险与依赖

### 主要风险
- Windows 桌面图标布局接口兼容性复杂
- 分辨率变化导致布局恢复偏移
- 文件监听与用户手动操作存在冲突
- 托盘和 Explorer 生命周期耦合

### 关键依赖
- Win32 / Shell API 可用性
- Windows Explorer 行为稳定性
- 本地文件系统权限正常
