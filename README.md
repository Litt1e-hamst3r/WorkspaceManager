# Workspace Manager

一个给 Windows 用户用的桌面管理工具。

它把一些原本很零碎的操作收进一个地方：

- 隐藏 / 显示桌面图标
- 隐藏 / 显示任务栏
- 保存桌面布局
- 恢复桌面布局
- 用模式快速切换不同桌面状态
- 最小化到托盘后台运行

如果你经常需要录屏、演示、开会共享屏幕，或者只是单纯不想让桌面越来越乱，这个工具就是为这种场景做的。

---

## 这是什么

`Workspace Manager` 是一个面向 Windows 的轻量桌面管理工具。

它不是文件同步工具，也不是桌面美化壳层。  
它主要解决的是下面这些高频小问题：

- Windows 原生的“隐藏桌面图标”入口太深
- 桌面图标位置一乱，很难恢复
- 演示前想快速让桌面干净一点
- 程序缩到托盘后，希望还能用快捷键一把拉回来

---

## 现在能做什么

当前版本已经可以完成以下事情：

### 桌面状态控制

- 一键显示 / 隐藏桌面图标
- 一键显示 / 隐藏任务栏
- 刷新并查看当前状态

### 托盘与窗口行为

- 程序常驻托盘
- 最小化到托盘
- 关闭窗口不退出
- 双击托盘图标恢复主窗口
- 通过全局快捷键恢复主窗口

### 布局管理

- 保存当前桌面图标布局
- 保存时自动生成预览图
- 从列表查看布局缩略图
- 点击查看大图预览
- 恢复指定布局
- 删除旧布局

### 模式系统

- 预设模式：
  - 默认模式
  - 工作模式
  - 演示模式
- 自定义模式：
  - 新建
  - 编辑
  - 删除
- 模式可绑定布局
- 启动时自动应用默认模式

### 设置项

- 开机自启
- 启动时最小化到托盘
- 最小化时隐藏到托盘
- 关闭窗口时隐藏到托盘
- 自定义桌面图标切换快捷键
- 自定义显示主窗口快捷键
- 支持按键捕获式录入快捷键

## 适合谁

如果你符合下面任意一种情况，这个工具会比较有用：

- 经常录屏、演示、共享桌面
- 桌面图标经常被分辨率变化打乱
- 想把“切换桌面状态”做得更快
- 不喜欢系统自带入口太绕
- 希望常用桌面控制放到托盘里

## 怎么用

### 1. 隐藏桌面图标

打开程序后，在“概览”页可以直接切换：

- 显示桌面图标
- 隐藏桌面图标

也可以通过你在设置页里配置的全局快捷键快速切换。

### 2. 隐藏任务栏

同样在主界面可直接切换任务栏显示状态。  
适合录屏、演示、共享屏幕时临时使用。

### 3. 保存布局

切到“布局”页：

1. 输入布局名称
2. 点击保存
3. 程序会记录当前桌面图标位置
4. 同时生成一张预览图，方便后面识别

### 4. 恢复布局

在布局列表中选择一个已保存布局后：

1. 查看预览
2. 点击“恢复布局”
3. 程序会尝试把当前桌面图标恢复到保存时的位置

### 5. 使用模式

模式适合做“整套状态切换”：

- 工作模式：适合专注，通常隐藏桌面图标
- 演示模式：适合会议或录屏，尽量减少干扰
- 自定义模式：按自己的习惯组合图标、任务栏和布局

你也可以把某个模式设成默认模式，让程序启动后自动切过去。

### 6. 找回主窗口

如果你：

- 把程序最小化到托盘了
- 隐藏了任务栏
- 一下子找不到程序窗口了

可以直接按你设置的“显示主窗口快捷键”把窗口拉回来。

## 数据保存在哪里

程序数据默认保存在当前用户的本地应用数据目录：

`%LOCALAPPDATA%\WorkspaceManager`

主要包括：

- `settings.json`：程序设置
- `modes.json`：模式配置
- `layouts\`：布局快照
- `layout-previews\`：布局预览图

这意味着：

- 重装程序前可以备份这些文件
- 换机时也可以手动迁移

## 当前已知限制

这个项目当前仍然是 MVP，下面这些点需要提前知道：

- 桌面布局恢复依赖 Windows Explorer 的桌面 ListView 行为
- 在不同分辨率、多显示器、系统缩放场景下，布局恢复可能存在偏差
- 如果桌面图标对应文件已经不存在，恢复时不会完全一致
- 当前还没有桌面整理规则、自动整理、撤销整理这些能力
- 当前以手工验证为主，自动化测试还在逐步补齐
